Introduction

Shiny is an exciting R library for interactive visualisation of data, using plots, figures and tables that the user can modify.

The current blog presents a Shiny App for handwritten digits. This blog is not meant to be a tutorial on learning Shiny, for which the following reference is more appropriate [1]. Rather it is meant to be a case example how Shiny can be used in imaging analysis of handwritten digits, including inspection and evaluation of digits. No prior knowledge of Shiny is assumed in order to follow the instructions in this blog.

Loading the datasets

In the current context, a selection of data from the MNIST will be used. MNIST is a famous set of handwritten digits [2]. In addition, a self-made set of handwritten digits will be used [3].

To limit required space and loading time, I have created a set of 200 digits for each set, which are now at my GitHub page (see below the link).

If you want to run my App locally on your workstation or laptop, you could download the selected data using the following code:

myurl<-"https://raw.githubusercontent.com/RRighart/App-Digits/master/selected-data.csv"

download.file(myurl,z,mode="wb")
train1 <- read.csv(z)
file.remove(z)

Running the App

If you do not have Shiny yet, you need to install it using the code install.packages("shiny"). Then type in library(shiny).

library(shiny)

The App can be run in your local space. You need to set your path directory for pa, for example pa="/home/my-directory/Shiny/". When you have set the directory, create under this directory a folder with the name App-Digits.

library(shiny)
setwd(pa)
runApp("App-Digits")

About the App

When we have run the App by `runApp(App-Digits)`, we will have the following:

The App shows an interface in which one can scroll through images of handwritten digits by hitting the button Next Image. The index simply indicates the rownumber of the dataset that is displayed. By selecting one of the pixels, it is possible to obtain the pixel coordinate (in the raster of 28 by 28 pixels) and pixel intensity.

On the right, the App shows a selection menu with radio buttons. Here an evaluator could indicate which digit is displayed. This is very useful if you want your image set to be validated, for example human evaluators for machine learning research. Below, one can select between different datasets, in this case the MNIST or Own handwritten digits.

library(shiny)

ui <- fluidPage(
  tags$style(type="text/css", "plot img{float:left;}"),
  
  h3("User interface for evaluating digits"),
  h6("In this interface you are requested to evaluate the label of various handwritten digits"),
  
  fluidRow(
   
    column(6, offset = 0,
      h4("Image Display"),
      plotOutput("view", height=300, click = "plot_click")
      ,
    fluidRow(  
      column(4, offset = 0,
      h6("Press button to forward"),
      actionButton("counter", label= "Next Image", icon("paper-plane"), style="color: #fff; background-color: #337ab7; border-color: #2e6da4"),
      h6("Index"),

      ),
      column(4, offset=0,
      h6("Pixel coordinates and pixel intensity"),

server <- function(input, output) {

 output$value <- renderText({
    input$counter
    })

 output$value2 <- renderText({
    input$radio
    })
 
 datasetInput <- reactive({
    switch(input$dataset,
        "MNIST" = train1,
        "Own Digits" = train2)
    })
 
 output$view <- renderPlot({
    dataset <- datasetInput()
    m = t(apply(matrix(unlist(dataset[input$counter+1,-1]), nrow=28, byrow=TRUE), 2, rev))
    par(mfrow=c(1,1),
        oma = c(0,0,0,0) + 0.1,
        mar = c(2,0,2,4) + 0.1)
    image(m, col=grey.colors(255), axes=FALSE, asp=1)
    })

 output$info <- renderText({
    dataset <- datasetInput()
    x=as.numeric(input$plot_click$x)
    y=as.numeric(input$plot_click$y)
    nx=ifelse(floor((x+0.03)*28)>0, floor((x+0.03)*28), 1)
    ny=ifelse(floor((y+0.03)*28)>0, floor((y+0.03)*28), 1)
    m = t(apply(matrix(unlist(dataset[input$counter+1,-1]), nrow=28, byrow=TRUE), 2, rev))
    paste0("X coord.= ", round(nx,2), "\nY coord.= ", round(ny,2), "\nPix int.= ", round(m[nx,ny],2))
}

shinyApp(ui, server)

Closing words

This is just a short example how Shiny can be used to display and collect data using an interactive interface. The interactive Shiny approach can be easily applied to any image or dataset that needs to be used or evaluated. Other functionalities can be added to Shiny Apps as well. Say for example that you or a coworker wanted to inspect handwritten digits for a certain label category. In Shiny it would be possible to add a menu, from which a label category from 0-9 could be selected, in order to only display digits from that category.

Shiny could be very useful not only for handwritten digits but any domain that uses image evaluations, such as crop classification, or medical image classification.
